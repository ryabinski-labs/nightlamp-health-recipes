# Maintainers

This file is the source of truth for who can merge to the default branch.
The list is mirrored to GitHub repository permissions.

Maintainers can merge pull requests to `main` after required CI checks pass.
A blanket approving review is not required. Non-maintainers cannot merge
even if checks pass.

## Becoming a maintainer

Open a discussion (or an issue if discussions are disabled) proposing a new
maintainer. The current lead maintainer decides. Criteria considered:

- Demonstrated contributions: merged recipes, reviews, or sustained triage.
- Familiarity with the recipe format and the audience this repo serves.
- Willingness to participate in the quarterly recipe review.

## Stepping down

Email the lead maintainer or open a PR removing yourself from this list.
